About 🪜 LITRA

Workflow software for translation, post-editing, and review datasets

🪜 LITRA helps teams turn structured source data into reviewed multilingual translation datasets. It is built for practical data work: importing varied file formats, assigning texts, reviewing changes, collecting comments, and exporting project data in formats that downstream research and engineering workflows can use.

Project Setup

Create projects from JSONL or JSON, auto-detect schema keys, and map source text, source language, identifiers, instructions, seed translations, and translated instructions.

Import Formats

Define reusable import profiles for different data structures, including RhaetoChat-style JSONL and Bouquet-style JSON with sentence lists.

Source Data

Review and edit source identifiers, source language, source text, instructions, and metadata. Creator links support source-level review before translation work continues.

Alternative Sources

Upload additional source-language versions for the same segment IDs. Translators can switch source-language tabs while working.

Language Management

Add and remove target languages, upload translations for new languages, import existing translations as submitted work or post-editing drafts, and track progress per language.

Translator Workspace

Public translator links automatically claim available texts, autosave drafts, support comments, source flags, source tabs, sentence-list editing, and adjustable source/target font size.

Review Workflows

Create project-level reviewer links, compare translations across languages, edit translations in fast review tables, mark items reviewed, and track QA warnings and comments.

Comments

Import comments from CSV by identifier or metadata field, merge multiple rows per document or segment, and keep discussion threads tied to target languages.

Manager Data Views

Use language-level and project-level translation data pages to filter, inspect, edit, and export all translation slots across a project.

Exports

Download JSONL, TXT ZIP packages, selected-language JSONL, and DOCX exports with source-language grids and translation-language grids.

Quality Signals

Basic QA warnings highlight empty translations and suspicious mismatches such as missing numbers. Managers and reviewers can filter by warning state.

Production Controls

Includes hot-query indexes, upload-size limits, simple rate limits for public links and uploads, and a production guard that requires PostgreSQL.

Typical Workflow

From source file to reviewed dataset

A manager imports source data, selects or adapts an import format, creates target languages, distributes translator links, reviews source issues and translation comments, then exports the filtered or complete dataset for release, analysis, or downstream model training.

1. Import Detect schema, map fields, and create project segments.
2. Translate Assign public links and collect drafts or submitted translations.
3. Review Edit, comment, flag source issues, and mark translations reviewed.
4. Export Download JSONL, TXT, or DOCX views for filtered or full results.
